## bx agents

### List agents

```bash
bx agents list
```

Shows all your agents with name, runtime, model, and status.

### Create an agent

```bash
bx agents create <name> [options]
```

| Option | Description | Default |
|--------|-------------|---------|
| `-r, --runtime <runtime>` | Runtime: `claude-code`, `opencode`, or `openclaw` | `claude-code` |
| `-m, --model <model>` | Model override | Runtime default |
| `-i, --instructions <text>` | System instructions for the agent | None |
| `-t, --type <type>` | Agent type: `hosted` or `byoa` | `hosted` |
| `-d, --directory <path>` | Working directory (BYOA only) | `~/.boxcrew/agents/<name>` |
| `--claude-path <path>` | Path to claude CLI binary (BYOA only) | `claude` |
| `--managed-api` | Route LLM calls through BoxCrew using credits (BYOA only) | Off (uses local API key) |

Agent names must be 3–30 characters, start with a letter, and contain only letters, numbers, and hyphens.

```bash
# Create a hosted agent
bx agents create my-agent
bx agents create research-bot --runtime opencode --model gpt-4o
bx agents create reviewer --instructions "Review code for security issues"

# Create a BYOA agent (uses your local API key by default)
bx agents create my-local-agent --type byoa

# Create a BYOA agent that uses BoxCrew credits instead of local key
bx agents create my-local-agent --type byoa --managed-api
```

<Info>
**BYOA API billing**: By default, BYOA agents use your local API key (e.g. your own Anthropic key). With `--managed-api`, LLM calls are routed through BoxCrew's proxy and billed to your BoxCrew credit balance — same as hosted agents.
</Info>

### Update an agent

```bash
bx agents update <agent> [options]
```

| Option | Description |
|--------|-------------|
| `--name <name>` | Rename the agent |
| `-m, --model <model>` | Change model |
| `-i, --instructions <text>` | Change instructions |
| `--managed-api` | Enable managed API (route LLM calls through BoxCrew, uses credits) |
| `--no-managed-api` | Disable managed API (use own API key) |

```bash
# Rename an agent
bx agents update my-agent --name new-name

# Change model
bx agents update my-agent --model claude-sonnet-4-5-20250514

# Switch a BYOA agent to BoxCrew credits
bx agents update my-local-agent --managed-api

# Switch back to own API key
bx agents update my-local-agent --no-managed-api
```

<Info>
For BYOA agents, changes to `model` or `managed_api` will automatically reconnect the local daemon with the new configuration.
</Info>

### Chat with an agent

```bash
bx agents chat <name> <message>
```

Sends a message and streams the response. The agent can write files, run commands, and perform actions inside its VM while responding.

```bash
bx agents chat my-agent "Write tests for the auth module"
```

### Delete an agent

```bash
bx agents delete <name>
```

Permanently destroys the agent and its VM, including all files.

## bx agents connect / disconnect

Connect your own machine as a BoxCrew agent (BYOA — Bring Your Own Agent). The agent runs locally and connects to BoxCrew via WebSocket so you can interact with it from the dashboard or API.

### Connect a local agent

```bash
bx agents connect <agent-name> [options]
```

| Option | Description | Default |
|--------|-------------|---------|
| `-d, --dir <path>` | Working directory for the agent | `~/.boxcrew/agents/<name>` |
| `--claude-path <path>` | Path to the `claude` CLI binary | `claude` |
| `--all` | Reconnect all previously connected agents | |

Spawns a background daemon that connects to BoxCrew. If no `--dir` is specified, a default workspace is automatically created at `~/.boxcrew/agents/<name>/`.

State files are stored in `~/.boxcrew/`:

- `{agentId}.meta.json` — connection metadata (PID, cwd, runtime)
- `{agentId}.log` — daemon logs
- `{agentId}.pid` — process ID

```bash
# Connect with default workspace
bx agents connect my-local-agent

# Connect using an existing project directory
bx agents connect my-local-agent --dir ~/code/my-project

# Reconnect all previously connected agents
bx agents connect --all
```

### Disconnect a local agent

```bash
bx agents disconnect <agent-name>
```

Stops the background daemon for the given agent.

```bash
bx agents disconnect --all
```

### Show local agent status

```bash
bx status
```

Lists all locally connected agents with their status (online/offline), runtime, working directory, and PID.

## bx keys

### List keys

```bash
bx keys list
```

### Create a key

```bash
bx keys create [options]
```

| Option | Description |
|--------|-------------|
| `-n, --name <name>` | Name for the key |
| `-e, --expires <date>` | Expiration date (ISO 8601) |

```bash
bx keys create --name "CI/CD"
bx keys create --name "Temp" --expires "2026-12-31T00:00:00Z"
```

<Warning>The full key (`bxk_...`) is shown only once. Store it somewhere safe.</Warning>

### Revoke a key

```bash
bx keys revoke <id>
```

## bx api

Make raw API requests. Works like `gh api`.

```bash
bx api <method> <path> [options]
```

| Option | Description |
|--------|-------------|
| `-d, --data <json>` | JSON request body |
| `-H, --header <header...>` | Additional headers (`key:value`) |

```bash
# List agents
bx api GET /agents

# Create an agent
bx api POST /agents -d '{"name":"test","runtime":"claude-code"}'

# Get billing info
bx api GET /billing

# Check install status
bx api GET /agents/my-agent/install-status

# Delete an agent
bx api DELETE /agents/my-agent
```
