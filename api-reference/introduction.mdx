## Base URL

```
https://api.boxcrew.ai
```

## Authentication

Authenticate with an API key in the `Authorization` header. Create keys from the [dashboard](https://boxcrew.ai) or with [`bx keys create`](/cli/commands#create-a-key).

```bash
curl https://api.boxcrew.ai/agents \
  -H "Authorization: Bearer bxk_your_key_here"
```

## Streaming (SSE)

`POST /agents/{name}/chat` returns a [Server-Sent Events](https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events) stream. Use `curl -N` or an SSE client to consume it.

```bash
curl -N -X POST https://api.boxcrew.ai/agents/my-agent/chat \
  -H "Authorization: Bearer $BOXCREW_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello"}'
```

Events follow the [OpenAI Responses API](https://platform.openai.com/docs/api-reference/responses) format:

```
data: {"type":"response.created","response":{...}}
data: {"type":"response.in_progress"}
data: {"type":"response.output_text.delta","delta":"Hello! "}
data: {"type":"response.output_text.delta","delta":"How can I help?"}
data: {"type":"response.completed","response":{...}}
```

Key event types:

| Event | Description |
|-------|-------------|
| `response.created` | Response initialized |
| `response.in_progress` | Streaming started |
| `response.output_text.delta` | Text chunk from the agent |
| `response.reasoning_text.delta` | Thinking/reasoning chunk |
| `response.function_call_arguments.delta` | Tool call arguments chunk |
| `response.output_item.done` | An output item completed |
| `response.completed` | Response finished |
| `response.failed` | Response failed with error |

### Reconnecting to a stream

If your connection drops, reconnect with `GET /agents/{name}/chat/stream?since=N` where `N` is the last `sequence_number` you received. Events with a sequence number ‚Č§ `N` are skipped.

### Request body

```json
{
  "message": "Your message to the agent",
  "filePaths": ["/path/to/file"],
  "fileMetadata": [{"filename": "data.csv", "mediaType": "text/csv"}]
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `message` | string | Yes | The message to send |
| `filePaths` | string[] | No | Paths to files on the agent's VM to include as context |
| `fileMetadata` | object[] | No | Metadata for uploaded files |

## Errors

```json
{
  "statusCode": 400,
  "message": "Agent name must be 3-30 characters",
  "error": "Bad Request"
}
```

| Status | Meaning |
|--------|---------|
| `400` | Invalid parameters |
| `401` | Missing or invalid auth |
| `402` | Insufficient credits |
| `403` | Access denied |
| `404` | Not found |
| `409` | Resource already exists |
| `500` | Server error |
